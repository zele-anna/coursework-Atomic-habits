# Приложение для отработки полезных привычек
## Цель проекта
Разработка приложения для отработки полезных привычек

### Установка и настройка
Все необходимые зависимости указаны в файле [pyproject.toml](pyproject.toml)
Для работы необходимо создать базу данных PostgreSQL, а также Телеграм-бота и прописать переменные окружения в файле .env по примеру [.env.example](.env.example)

### Запуск программы
Требуется запустить сервер командой python manage.py runserver

### Использование функций и примеры работы
Все эндпоинты закрыты для неавторизованных пользователей, исключения - эндпоинты для регистрации пользователей, входа, а также списка публичных привычек (is_public=True).
1. Регистрация пользователей производится по модели User, содержащей поля:
- email (логин);
- телеграм chat_id (опционально).
2. Зарегистрированный пользователь имеет возможность создавать привычки. Модель Habit содержит поля:
- owner - "Пользователь" (присваивается автоматически по пользователю, от которого осуществлен логин) 
- action - "Действие"
- time_to_start - "Время начала"
- place - "Место выполнения"
- is_enjoyable - "Признак приятной привычки" (по умолчанию False)
- associated_habit - "Связанная привычка"
- periodicity - "Периодичность" (по умолчанию 1)
- reward - "Вознаграждение"
- time_to_complete - "Время на выполнение" (по умолчанию 60)
- is_public - "Признак публичности" (по умолчанию False)

При создании привычки осуществляется ее привязка  к текущему пользователю в качестве владельца.
После создания привычки владелец имеет права на CRUD-операции по ней.
Пользователь имеет права на работу только со своими привычками.
Эндпоинты работы с привычками реализованы через Viewsets.
Для привычки имеется единственный сериализатор HabitSerializer с выводом всех полей модели.

3. Валидаторы:
- Исключен одновременный выбор связанной привычки (associated_habit) и указания вознаграждения (reward).
- Время выполнения (time_to_complete) должно быть не больше 120 секунд.
- В связанные привычки могут попадать только привычки с признаком приятной привычки (is_enjoyable).
- У приятной привычки не может быть вознаграждения (reward) или связанной привычки (associated_habit).
- Нельзя выполнять привычку реже, чем 1 раз в 7 дней (periodicity).

4. Пагинация. Для вывода списка привычек используется пагинация CustomPagination с выводом по 5 объектов на страницу.

5. Интеграция. 
Для пользователей, у которых указан Телеграм chat_id производится отправка уведомлений в чат-бот с напоминанием о выполнении привычек с помощью create_periodic_task() в [tasks.py](habits/tasks.py).  

6. Безопасность. Для проекта настроен CORS.

7. Документация. Для реализации экранов силами фронтенд-разработчиков настроен вывод документации.

### Тестирование
